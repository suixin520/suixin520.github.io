Hexo搭建个人博客详细过程

1. 环境配置：

  1. 下载安装git;
  1. 下载安装Node.js; //最好安装稳定版的Node
  1. 申请github账户; //如果已有，请飘过此句

2. hexo安装：

  • 新建一个文件夹用来保存本地博客文件,在该文件夹下执行：

    npm install -g hexo-cli //npm包全局安装hexo，这个过程需要等待一段时间...

  • 等上面执行完成后再执行：

    hexo init <folder> //如果自己在外面新建好了文件夹，<folder>可以不要，下载hexo包

cd <folder>//如果上面没有folder这句可以不要

npm install//等待（漫长的）一段时间...

  • 安装完成后可以执行下面命令：

    hexo server

    然后可以在浏览器中打开localhost:4000，就能看到你的本地静态blog网页了。

3. 配置github，把代码托管到github上

  • 首先登陆你的github，然后新建一个仓库，名字最好是这样的格式：yourname.github.io (yourname指的是你github的name)。

  • 然后更改本地的_config.yml文件，用记事本（任意编辑器都可）打开，翻到最下面的deploy改为：

    deploy:
      type: git
      repository: https://github.com/yourname/yourname.github.io.git
      branch: master

  • 然后在刚刚的下载好hexo文件夹中右键，git bush,在黑窗口中输入：

    hexo generate

//hexo g完成后再输入：

hexo deploy

    如果提示not found git错误：

1. 首先检查本地的_config.yml文件格式是否正确，每个冒号后面有个空格，type是git而不是github，地址要输入正确
2. 这些都正确如果还报错输入下面一行代码：

        npm install hexo-deployer-git --save

    然后再generate和deploy

  • 这些过程成功完成后就可以在你的仓库中看到你上传上的代码了，然后就可以访问yourname.github.io看到你的个人博客页面了。这个看起来和你在本地localhost:4000看的页面是一样的，但是你可以在任何一台有网络的电脑上看到你的页面。

    小提示：当你代码上传到github上之后，你立即访问的话可能会出现404错误，这个是正常的，大概等一段时间之后，快则一两分钟，慢则

半个小时以上（看人品）。所以不用担心这个问题，只要代码上传到你的仓库了。
