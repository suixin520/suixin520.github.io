Node.js-url模块

1.区分一个简单的概念

URI: 统一资源标识符；它是一种语义上的抽象概念，是一种字符串格式的规范。简单来说就是纯粹的句法结构。比如mailto: suixin@host.com。

URL: 统一资源定位符；是一个结构化字符串，是URI的一种，支持解析的语法运算以及查找主机和打开指定资源的连接之类的网络I/O操作。简单说就是用哪种协议去访问一个资源。比如https://www.baidu.com。

url.format(obj)

  format方法可以将一个对象形式转换为一个url地址。

得到的结果如下：

‘https://suixinblog.com/2017/03/26/Node.js‘

格式化过程如下：

  • 创建一个新的空字符串result
  • 如果obj.protocol是一个字符串则会添加到result，如果不是则抛出Error
  • 如果obj.slashes属性为真或者obj.protocol以http/https/ftp/gopher/file开头，则//会被添加到result
  • 如果obj.auth属性为真，且obj.host或obj.hostname不为undefined，则obj.auth会被添加到result，后面带上@
  • 如果obj.host属性为undefined，则如果obj.hostname是一个字符串，它会被添加到result，否则抛出Error，如果obj.port属性为真，obj.hostname不为undefined，则：会被加到result，obj.port的值会被加到result
  • 如果ojb.host属性值为真，则url.host的值会被加到result
  • 如果url.pathname属性是一个字符串不是以/开头，则/会被加到result，url.pathname的值会被加到result，否则为undefined或者不是一个字符串，则抛出Error
  • 如果url.search属性为undefined且url.query属性是一个object，则？会被添加到result，后面跟上url.query的值传入querystring模块的stringify()方法的调用结果，否则如果obj.search是一个字符串，如果没有？，则？加到result，url.search值也会被加到result。如果url.search不是undefined也不是字符串，则抛出Error
  • 如果url.hash是一个字符串，没有#开头则#会被加到result，url.hash的值会被加到result，如果url.hash的值不是一个字符串也不是一个undefined，则抛出Error
  • 返回result。
